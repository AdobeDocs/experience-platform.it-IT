# Dizionario campo XDM


[!DNL Experience Data Model] (XDM) definisce un set di campi standard che possono essere utilizzati durante la composizione di schemi di Adobe Experience Platform. L&#39;utilizzo di questi campi determina un comportamento coerente tra i servizi [!DNL Platform], in quanto tutti i servizi che utilizzano XDM comprenderanno esplicitamente le definizioni specifiche fornite da questi campi.

Consulta le [nozioni di base sulla composizione dello schema](composition.md) per ulteriori informazioni sui blocchi predefiniti degli schemi, inclusi i principi e le best practice per la definizione degli schemi per [!DNL Experience Platform].

La tabella seguente include il nome del campo (non tutti i campi XDM sono namespace utilizzando &quot;xdm&quot;) insieme al titolo e alla descrizione, al tipo e al formato, se necessario, nonché il tipo XDM corrispondente e le risorse (schemi, gruppi di campi di schema e tipi di dati) in cui è possibile trovare il campo.

È possibile utilizzare la funzionalità di ricerca in [!DNL Schema Editor] per visualizzare i dettagli di queste risorse, come illustrato nell&#39;[esercitazione sull&#39;editor di schemi](../tutorials/create-schema-ui.md#browse).

>[!NOTE]
>
>Se un campo è contrassegnato come &quot;**OBSOLETO**&quot; significa che non deve più essere utilizzato nelle nuove implementazioni. Il campo continua a essere disponibile per compatibilità con le versioni precedenti, ma non sarà più supportato nelle versioni future.
