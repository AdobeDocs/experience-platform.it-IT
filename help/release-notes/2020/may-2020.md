## Aggiornamenti dell’interfaccia utente {#ux}

Adobe Experience Platform sta aggiornando il dominio e la barra di intestazione per migliorare la tua esperienza e unificarla ad altri [!DNL Experience Cloud] applicazioni.

- Un passaggio più semplice tra le organizzazioni o verso un’altra applicazione
- È stata migliorata la guida utente, inclusi gli articoli contenuti e la documentazione pertinente al contesto nel menu Guida
- Possibilità di fornire feedback sul [!DNL Experience Platform] e ticket di supporto per i file

L&#39;introduzione della nuova esperienza è graduale. Puoi visualizzare l’esperienza in [https://experience.adobe.com/platform](https://experience.adobe.com/platform).

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] utilizza l’apprendimento automatico e l’intelligenza artificiale per trarre informazioni dai tuoi dati. integrata in Adobe Experience Platform, [!DNL Data Science Workspace] consente di effettuare previsioni utilizzando il contenuto e le risorse di dati nelle soluzioni Adobe. Uno dei modi [!DNL Data Science Workspace] ciò avviene tramite l&#39;utilizzo di [!DNL JupyterLab]. [!DNL JupyterLab] è un&#39;interfaccia utente basata sul Web per [[!DNL Project Jupyter]](https://jupyter.org/) ed è strettamente integrato in Adobe Experience Platform. Offre agli scienziati dei dati un ambiente di sviluppo interattivo con cui lavorare [!DNL Jupyter] blocchi appunti, codice e dati.

**Nuove funzioni**

| Funzione | Descrizione |
|--- | ---|
| [!DNL JupyterLab Launcher] | La [!DNL JupyterLab Launcher] ora include gli avviatori per [!DNL Spark] 2.4 notebook. [!DNL Spark] Gli avviatori dei notebook 2.3 ora sono contrassegnati come obsoleti e sono impostati per essere rimossi in una versione successiva. |
| [!DNL Spark] 2,4 | Nuovo livello ([!DNL Spark]) e le ricette PySpark ora utilizzano [!DNL Spark] 2.4. |
| Kernel | Scala ([!DNL Spark]) i notebook vengono ora creati tramite il kernel Scala. I notebook PySpark ora vengono creati tramite l’ [!DNL Python] Crenale. La [!DNL Spark] e il kernel PySpark sono obsoleti e sono impostati per essere rimossi in una versione successiva. |
| Ricette | Nuovo PySpark e [!DNL Spark] ricette ora seguire le [!DNL Docker] un flusso di lavoro simile a [!DNL Python] e ricette R. |

Per ulteriori informazioni sulla migrazione dei notebook e delle ricette da utilizzare [!DNL Spark] 2.4. consultare il [guida all&#39;accesso ai dati per notebook](../../data-science-workspace/jupyterlab/access-notebook-data.md). Per informazioni più generali su Data Science Workspace, consulta la sezione [documentazione panoramica](../../data-science-workspace/home.md).

## [!DNL Destinations] {#destinations}

In [Real-time Customer Data Platform](../../rtcdp/overview.md), le destinazioni sono integrazioni predefinite con piattaforme di destinazione che attivano i dati per tali partner in modo semplice.

**[!DNL Facebook]**

Real-time CDP ora supporta l’attivazione dei dati per [!DNL Facebook], che consente di attivare i profili per [!DNL Facebook] campagne per il targeting del pubblico, la personalizzazione e la soppressione basate su e-mail con hash.

Per ulteriori informazioni sulle nuove funzionalità, consulta la sezione [Destinazione facebook](../../destinations/catalog/social/facebook.md) pagina.

**[!DNL Amazon Kinesis]e [!DNL Azure Event Hubs] destinazioni di archiviazione cloud in streaming**

Real-time CDP ora supporta l’attivazione dei dati per le destinazioni di archiviazione cloud in streaming, consentendoti di esportare i dati e gli eventi del pubblico in queste destinazioni in formato JSON. Puoi quindi descrivere la logica di business sopra questi eventi nelle tue destinazioni. Vedi sotto per i dettagli:

## [!DNL Real-time Customer Profile] {#profile}

Adobe Experience Platform ti consente di fornire ai clienti esperienze coordinate, coerenti e pertinenti, indipendentemente da dove e quando interagiscono con il tuo marchio. Con [!DNL Real-time Customer Profile], puoi visualizzare una visualizzazione olistica di ogni singolo cliente che combina dati provenienti da più canali, inclusi dati online, offline, CRM e di terze parti. [!DNL Profile] consente di consolidare i diversi dati dei clienti in una visualizzazione unificata che offre un account actionable e timestamp di ogni interazione con il cliente.

**Nuove funzioni**

| Funzione | Descrizione |
| -----------| ---------- |
| Nuove metriche di esportazione del profilo | Sono state aggiunte metriche ai processi di esportazione del profilo che mostrano il numero totale di profili esportati e il numero di profili in ogni spazio dei nomi. |
| Nuove metriche di Observability Insights | L’API di Observability Insights dispone ora delle metriche seguenti per lo streaming ingestion in [!DNL Profile]: Velocità di richiesta in entrata, tasso di acquisizione riuscito, dimensione del record ingestito. |
| Endpoint GET bulk | Aggiunto endpoint di GET in serie a [!DNL Real-time Customer Profile] API per abilitare il recupero di più risultati in una singola chiamata API. È ora possibile utilizzare in massa fino a 100 ID per le definizioni dei segmenti, i processi dei segmenti e i criteri di unione. |
| Sfoglia profilo per identità | In [!DNL Platform] Nell’interfaccia utente è ora possibile selezionare uno spazio dei nomi di identità e fornire un valore di identità per sfogliare un profilo. |

## Origini {#sources}

Adobe Experience Platform può acquisire dati da sorgenti esterne e allo stesso tempo strutturare, etichettare e migliorare tali dati utilizzando [!DNL Platform] servizi. È possibile acquisire dati da diverse sorgenti, come applicazioni di Adobe, archiviazione basata su cloud, software di terze parti e il sistema CRM in uso.

[!DNL Experience Platform] fornisce un’API RESTful e un’interfaccia utente interattiva che consente di impostare facilmente le connessioni sorgente per vari provider di dati. Queste connessioni di origine ti consentono di autenticare e connettersi a sistemi di archiviazione esterni e servizi CRM, impostare i tempi di esecuzione dell’acquisizione e gestire il throughput di inserimento dei dati.

| ------- | ----------- |
| Supporto aggiuntivo per API e interfaccia utente per i sistemi di archiviazione cloud | Nuovi connettori sorgente per [!DNL Azure File Storage]. |
| Supporto aggiuntivo per API e interfaccia utente per i database | Nuovi connettori sorgente per [!DNL Azure Data Explorer], IBM DB2 e [!DNL Oracle DB]. |
| Adobe Audience Manager a [!DNL Experience Platform] condivisione dei dati | Il processo di provisioning per [!DNL Audience Manager] connettore aggiornato. [!DNL Audience Manager] set di dati per [!DNL Real-time Customer Profile] sono ora disattivate per impostazione predefinita. Puoi scegliere manualmente i set di dati da promuovere [!DNL Profile]. Le nuove impostazioni predefinite non sono retroattive e influiscono solo sul provisioning per le nuove [!DNL Audience Manager] connettori Per ulteriori informazioni, consulta la sezione [Guida utente per i set di dati](../../catalog/datasets/user-guide.md). |
