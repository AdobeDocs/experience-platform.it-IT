## Aggiornamenti dell’interfaccia utente {#ux}

Adobe Experience Platform sta aggiornando il dominio e la barra di intestazione per migliorare la tua esperienza e unificarla ad altre applicazioni [!DNL Experience Cloud].

- Un passaggio più semplice tra le organizzazioni o verso un’altra applicazione
- È stata migliorata la guida utente, inclusi gli articoli contenuti e la documentazione pertinente al contesto nel menu Guida
- Possibilità di fornire feedback sui [!DNL Experience Platform] ticket di supporto file e

L&#39;introduzione della nuova esperienza è graduale. Per visualizzare l&#39;esperienza, visita [https://experience.adobe.com/platform](https://experience.adobe.com/platform).

## [!DNL Data Science Workspace] {#dsw}

[!DNL Data Science Workspace] utilizza l’apprendimento automatico e l’intelligenza artificiale per trarre informazioni dai tuoi dati. Integrato in Adobe Experience Platform, [!DNL Data Science Workspace] consente di fare previsioni utilizzando i contenuti e le risorse dati nelle soluzioni Adobe. Uno dei modi in cui [!DNL Data Science Workspace] esegue questa operazione è l&#39;utilizzo di [!DNL JupyterLab]. [!DNL JupyterLab] è un’interfaccia utente basata su web per  [[!DNL Project Jupyter]](https://jupyter.org/) ed è strettamente integrata in Adobe Experience Platform. Fornisce un ambiente di sviluppo interattivo per consentire agli scienziati dei dati di lavorare con notebook, codice e dati [!DNL Jupyter].

**Nuove funzionalità**

| Funzione | Descrizione |
|--- | ---|
| [!DNL JupyterLab Launcher] | Il [!DNL JupyterLab Launcher] ora include avviatori per i notebook [!DNL Spark] 2.4. [!DNL Spark] Gli avviatori dei notebook 2.3 ora sono contrassegnati come obsoleti e sono impostati per essere rimossi in una versione successiva. |
| [!DNL Spark] 2,4 | Le nuove ricette Scala ([!DNL Spark]) e PySpark ora utilizzano [!DNL Spark] 2.4. |
| Kernel | I notebook Scala ([!DNL Spark]) vengono ora creati tramite il kernel Scala. I notebook PySpark ora vengono creati tramite il kernel [!DNL Python] . Il kernel [!DNL Spark] e PySpark sono obsoleti e sono impostati per essere rimossi in una versione successiva. |
| Ricette | Le nuove ricette PySpark e [!DNL Spark] seguono ora il flusso di lavoro [!DNL Docker] simile alle ricette [!DNL Python] e R. |

Per ulteriori informazioni sulla migrazione dei notebook e delle ricette per utilizzare [!DNL Spark] 2.4, consultare la [guida all&#39;accesso ai dati del notebook](../../data-science-workspace/jupyterlab/access-notebook-data.md). Per informazioni più generali su Data Science Workspace, consulta la [documentazione di panoramica](../../data-science-workspace/home.md).

## [!DNL Destinations] {#destinations}

In [Real-time Customer Data Platform](../../rtcdp/overview.md), le destinazioni sono integrazioni predefinite con piattaforme di destinazione che attivano i dati per tali partner in modo semplice.

**[!DNL Facebook]**

Real-time CDP ora supporta l’attivazione dei dati su [!DNL Facebook], consentendoti di attivare profili per le campagne [!DNL Facebook] per il targeting di pubblico, la personalizzazione e la soppressione in base a e-mail con hash.

Per ulteriori informazioni sulla nuova funzionalità, consulta la pagina [Destinazione Facebook](../../destinations/catalog/social/facebook.md) .

**[!DNL Amazon Kinesis]e destinazioni di archiviazione  [!DNL Azure Event Hubs] in streaming cloud**

Real-time CDP ora supporta l’attivazione dei dati per le destinazioni di archiviazione cloud in streaming, consentendoti di esportare i dati e gli eventi del pubblico in queste destinazioni in formato JSON. Puoi quindi descrivere la logica di business sopra questi eventi nelle tue destinazioni. Vedi sotto per i dettagli:

## [!DNL Experience Platform Web SDK] ed [!DNL Experience Platform Edge Network] {#edge}

I [!DNL Experience Platform Web SDK] e [!DNL Experience Platform Edge Network] consentono agli utenti di inviare dati ad Adobe Experience Platform e ad altre soluzioni Adobe in tempo reale per i dispositivi e i browser degli utenti finali. L&#39;elenco più recente dei casi d&#39;uso si trova nella nostra [roadmap pubblica](https://github.com/adobe/alloy/projects/5) che viene aggiornata spesso.

**Nuove funzionalità**

| Funzione | Descrizione |
|--- | ---|
| Supporto per ECID | L&#39;SDK supporta ECID out of the box senza ulteriori librerie o informazioni da installare. |
| Interfaccia utente di configurazione | Per gestire le impostazioni dell’ID di configurazione con la nuova interfaccia utente di configurazione perimetrale in [!DNL Launch], è necessario essere nell’elenco Consentiti per poter accedere. |
| Combinazione di Adobe Experience Platform Web SDK | Un mixin da utilizzare con [!DNL Experience Platform web SDK] che include tutti i campi supportati. |
| Controlli sul consenso al corso | Fornisce alle aziende il controllo sull&#39;opt-in e la rinuncia a [!DNL Experience Platform Web SDK]. |
| Supporto del debug lato client nel nuovo [!DNL Experience Cloud Debugger Extension] | Vedi le richieste di [!DNL Experience Platform web SDK] e le tracce edge per vedere come i dati scorrono nel sistema. |
| Adobe Analytics | Invia dati alle suite di rapporti [!DNL Analytics] tramite la configurazione perimetrale. XDM viene appiattito nei dati contestuali e supporta l’assegnazione di tag a più suite. |
| Adobe Target | Supporto per Adobe Target. Compreso Compositore esperienza visivo, Compositore esperienza visivo basato su moduli, A/B, XT, Personalizzazione automatizzata, MVT. |
| Supporto di Adobe Audience Manager | Supporto per le sincronizzazioni [!DNL Audience Manager] ID, le destinazioni URL e le destinazioni cookie. |

| Generatore di oggetti XDM | Nell’estensione launch è ora possibile creare oggetti XDM come elementi dati. |

Per ulteriori informazioni su [!DNL Platform Web SDK] e [!DNL Edge Network], consulta la [documentazione](../../edge/home.md).

## [!DNL Real-time Customer Profile] {#profile}

Adobe Experience Platform ti consente di promuovere esperienze coordinate, coerenti e rilevanti per i clienti, indipendentemente da dove e quando interagiscono con il tuo marchio. Con [!DNL Real-time Customer Profile] puoi visualizzare una visualizzazione olistica di ogni singolo cliente che combina dati provenienti da più canali, inclusi dati online, offline, CRM e di terze parti. [!DNL Profile] consente di consolidare i diversi dati dei clienti in una visualizzazione unificata che offre un account actionable e timestamp di ogni interazione con il cliente.

**Nuove funzionalità**

| Funzione | Descrizione |
| -----------| ---------- |
| Nuove metriche di esportazione del profilo | Sono state aggiunte metriche ai processi di esportazione del profilo che mostrano il numero totale di profili esportati e il numero di profili in ogni spazio dei nomi. |
| Nuove metriche di Observability Insights | L’API di Observability Insights dispone ora delle metriche seguenti per lo streaming ingestion in [!DNL Profile]: Velocità di richiesta in entrata, tasso di acquisizione riuscito, dimensione del record ingestito. |
| Endpoint GET in blocco | È stato aggiunto l’endpoint GET di massa all’ API [!DNL Real-time Customer Profile] per abilitare il recupero di più risultati in una singola chiamata API. Ora è possibile utilizzare il metodo GET per 100 ID per le definizioni dei segmenti, i processi dei segmenti e i criteri di unione. |
| Sfoglia profilo per identità | Nell’interfaccia utente di [!DNL Platform] è ora possibile selezionare uno spazio dei nomi di identità e fornire un valore di identità per sfogliare un profilo. |

## Origini {#sources}

Adobe Experience Platform può acquisire dati da sorgenti esterne e allo stesso tempo strutturare, etichettare e migliorare tali dati utilizzando i servizi [!DNL Platform] . Puoi acquisire dati da diverse sorgenti, come applicazioni Adobe, archiviazione basata su cloud, software di terze parti e il tuo sistema CRM.

[!DNL Experience Platform] fornisce un’API RESTful e un’interfaccia utente interattiva che consente di impostare facilmente le connessioni sorgente per vari provider di dati. Queste connessioni di origine ti consentono di autenticare e connettersi a sistemi di archiviazione esterni e servizi CRM, impostare i tempi di esecuzione dell’acquisizione e gestire il throughput di inserimento dei dati.

**Nuove funzionalità**

| Funzione | Descrizione |
| ------- | ----------- |
| Supporto aggiuntivo per API e interfaccia utente per i sistemi di archiviazione cloud | Nuovi connettori sorgente per [!DNL Azure File Storage]. |
| Supporto aggiuntivo per API e interfaccia utente per i database | Nuovi connettori sorgente per [!DNL Azure Data Explorer], IBM DB2 e [!DNL Oracle DB]. |
| Adobe Audience Manager alla condivisione dei dati [!DNL Experience Platform] | Il processo di provisioning per il connettore [!DNL Audience Manager] è stato aggiornato. [!DNL Audience Manager] i set di dati per  [!DNL Real-time Customer Profile] ora sono disabilitati per impostazione predefinita. Puoi scegliere manualmente i set di dati da promuovere in [!DNL Profile]. Le nuove impostazioni predefinite non sono retroattive e influiscono solo sul provisioning per i nuovi connettori [!DNL Audience Manager]. Per ulteriori informazioni, consulta la [Guida utente ai set di dati](../../catalog/datasets/user-guide.md). |
